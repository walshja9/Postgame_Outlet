### Pregame feature contract

Every team state is higher-is-better and is updated only after its game row is emitted. A game row is `home_state - away_state`, plus neutral-aware home field and clipped rest difference. The initial candidate features are:

- Performance: PGO v0 pregame rating, passing EPA/play for and against, rushing EPA/play for and against, explosive-play rate for and against, sack rate for and against, giveaway rate, and takeaway rate.
- Quarterback: expected full-strength QB's shrunk EPA/dropback, CPOE, sack avoidance, ball security, rushing EPA/carry, log dropback sample, and draft/experience priors.
- Roster/coaching: returning offense and defense snap share, incoming prior snap share, rookie draft capital, head-coach continuity, and head-coach tenure.
- Availability: QB current-minus-full value, unavailable expected offense snap share, and unavailable expected defense snap share.
- Context: neutral-aware home indicator and `clip(home_rest - away_rest, -7, 7) / 7`.

Team performance uses an exponential state whose half-life is selected from `(4, 8, 16)` prior games inside chronological development folds. Rate denominators must be positive; otherwise the value is missing, never zero.

QB estimates use only earlier player games. Per-play metrics are shrunk toward the training-fold population mean with 200 dropbacks as the prior sample. The full-strength QB is the rostered QB with the highest pregame shrunk estimate. Current QB value applies the injury probability to that QB and the remaining probability to the best available replacement; this is the first explicit replacement-quality effect.

Historical injury probabilities are frozen as `Out=0.00`, `Doubtful=0.25`, `Questionable=0.70`, `Probable=1.00`, and no report status `=1.00`. `Note` is not a designation and falls back to practice status. When report status is blank or `Note`, the short and source-native `In Practice` labels map `Did Not Participate=0.70`, `Limited Participation=0.90`, and full participation or no row `=1.00`; `Out (Definitely Will Not Play)=0.00`. A recognized report designation takes precedence over practice status, and any other unknown nonblank status is an error. Expected unavailable unit share is prior four-game median snap share multiplied by `1 - availability_probability`.

Treat the weekly injury row as the final official pregame report. For 2013-2024 rows that expose `date_modified`, reject revisions timestamped after kickoff; the 2025 schema omits that optional field, which `source_audit.json` must record as a source limitation. Use the frozen release snapshot as the 2026 roster as-of boundary rather than inventing an unavailable transaction timestamp.

The full-strength state sets all rostered players to availability `1.0`. The current state changes only availability features and the expected QB mixture. Therefore, before centering:

```text
current-lineup rating = full-strength rating + availability adjustment
```

Center the 32 full-strength ratings to mean zero. Apply the same centering constant to current ratings; never recenter current ratings separately.

### Model and validation contract

- Fit a linear Huber-loss ridge model with NumPy iteratively reweighted least squares; do not add scikit-learn.
- Standardize and median-impute using training rows only. Append a missingness flag for each feature that is missing in the training fold. The intercept is not regularized.
- Candidate grid: team half-life `(4, 8, 16)`, ridge alpha `(1.0, 10.0, 100.0)`, Huber delta `(1.0, 1.5)`.
- Deterministic tie-break: lowest inner-fold MAE, then smallest `(half_life, alpha, delta)` tuple.
- Outer folds predict each season 2018 through 2025 after training only on earlier seasons. For each outer fold, select parameters on inner validation seasons from 2016 through the season before the outer fold, requiring at least three earlier training seasons.
- Refit the selected 2026 parameters using inner folds 2018-2025, then train through 2025 for the preseason snapshot.
- Compare the challenger and existing PGO v0 on the exact same `game_id` set. PGO v0 parameters remain its existing frozen 2002-2017 selection.
- Primary metric: MAE. Secondary metrics: median absolute error, RMSE, miss rates above 14 and 21 points, mean signed error, home/away signed error, season results, and fixed predicted-margin bands `(<-7, -7:-3, -3:3, 3:7, >7)`.
- Bootstrap 10,000 deterministic paired resamples of `(season, week)` blocks with seed `20260721`. Improvement is `v0 absolute error - challenger absolute error`; positive is better. Report percentile 95% confidence intervals.
- Frozen subgroups: changed/backup expected QB; availability adjustment `<= -1.5`; head-coach change; either team's returning snap share in that season's bottom quartile; weeks 1-4; weeks 5-18. A subgroup with fewer than 100 games is `INSUFFICIENT_EVIDENCE` and cannot independently pass or fail.
- Historical `PASS` requires all source/identity/leakage/reproducibility checks, both rating views for 32 teams, lower challenger MAE, aggregate improvement CI lower bound above zero, and no sufficient-evidence subgroup whose improvement CI upper bound is below zero.
